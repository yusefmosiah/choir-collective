## Core Transform Pattern

A general pattern for transforming documentation from one paradigm to another:

1. Collect Phase
- Gather related documents
- Group by category
- Map relationships
- Note patterns
- Recognize structure

2. Summarize Phase
- Extract key concepts
- Identify core patterns
- Map flows/relationships
- Note invariants
- Capture assumptions

3. Transform Phase
- Map from old paradigm
- Map to new paradigm
- Maintain coherence
- Enable evolution
- Preserve value

4. Refactor Phase
- Unify concepts
- Remove duplication
- Clarify hierarchy
- Strengthen patterns
- Improve flow

## Example: Web to Native Transform

Old Context (Web/React)
- Client-server architecture
- Browser extension wallet
- Web component patterns
- REST/GraphQL APIs
- Traditional documentation

New Context (Native/Swift)
- Event-driven architecture
- Actor-based isolation
- Local-first with chain authority
- Natural pattern emergence
- Value crystallization

